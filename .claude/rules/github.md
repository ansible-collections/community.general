# Github Rules

- Prefer using `gh` for all interactions with Github.
- Do not push any change unless explicitly requested.
- Do not run `git commit` nor `git push` unless explicitly requested.
- When creating an issue, you MUST use one of the templates in `.github/ISSUE_TEMPLATE`
  - Component name: one per line, use the relative path in the repo, one per line
  - Community.general version: if bug exists in `main` branch, report the version from galaxy.yml
  - Do generate the Code of Conduct checkbox - I am responsible for what you do, my child
- When writing GitHub comments or issue text that will be posted under the user's account,
  never use first-person language ("I", "my", "we"). Write in a neutral third-person tone.
